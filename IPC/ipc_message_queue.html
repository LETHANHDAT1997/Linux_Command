IPC — POSIX Message Queue

Message-based IPC với priority ordering: message quan trọng hơn được nhận trước, bất kể thứ tự gửi.

Message Queue — hàng đợi có ưu tiên
Kernel quản lý hàng đợi message. Sender gửi không cần receiver sẵn sàng. Message có priority — quan trọng hơn lên đầu queue.
Tại sao cần Message Queue khi đã có Pipe và Socket?
Pipe và Socket là byte stream hoặc connection-based: sender và receiver phải chạy đồng thời, không có khái niệm "message" riêng biệt, không có priority.

Message Queue giải quyết 3 vấn đề mà các IPC trước không làm được:
— Message boundary: mỗi mq_send() là 1 message trọn vẹn, mq_receive() luôn nhận đúng 1 message — không bao giờ bị tách.
— Priority ordering: message với priority cao hơn luôn được lấy ra trước, bất kể thứ tự gửi.
— Decoupled timing: sender gửi khi rảnh, receiver nhận khi rảnh — không cần connect trước.

POSIX vs SysV Message Queue
POSIX mq — dùng cái này
Header: <mqueue.h>
Tên: /myqueue (path-like)
Trả về mqd_t — là file descriptor thực!
Dùng được với select(), poll(), epoll()
Có mq_notify() — async notification
Dùng trong code mới
SysV msgq — legacy
Header: <sys/msg.h>
Dùng key_t (số nguyên), khó debug
Không phải fd — không dùng với select/poll
Không có async notification
Filter message theo msg_type

API chi tiết — từng tham số
mq_open(), mq_send(), mq_receive(), mq_close(), mq_unlink() — tất cả đều tương tự file API.
mq_open() — tạo hoặc mở queue
#include <mqueue.h> mqd_t mq_open(const char *name, int oflag); mqd_t mq_open(const char *name, int oflag, mode_t mode, struct mq_attr *attr); /* khi O_CREAT */
name
Tên queue, phải bắt đầu bằng /, không chứa / nào khác. Ví dụ: "/sensor_events". Queue được quản lý bởi kernel — nằm trong /dev/mqueue/ (nếu mount mqueue fs).
oflag
O_CREATTạo queue nếu chưa tồn tại. Cần thêm tham số mode và attr.
O_RDONLYChỉ nhận (mq_receive). Dùng ở phía consumer.
O_WRONLYChỉ gửi (mq_send). Dùng ở phía producer.
O_RDWRVừa gửi vừa nhận.
O_NONBLOCKNon-blocking: mq_send/receive trả về EAGAIN thay vì block khi queue đầy/rỗng.
O_EXCLKết hợp O_CREAT: lỗi EEXIST nếu đã tồn tại.
mode
Permission bits khi O_CREAT. 0600, 0660, 0666 — giống file.
attr
Con trỏ đến struct mq_attr để set thông số queue (xem trang 3). Truyền NULL để dùng giá trị mặc định của hệ thống.
Trả về: mqd_t (là file descriptor) thành công, (mqd_t)-1 lỗi. EEXIST, ENOENT, EMFILE
mq_send() — gửi message
#include <mqueue.h> int mq_send(mqd_t mqdes, const char *msg_ptr, size_t msg_len, unsigned int msg_prio); int mq_timedsend(mqd_t mqdes, const char *msg_ptr, size_t msg_len, unsigned int msg_prio, const struct timespec *abs_timeout);
mqdes
Descriptor từ mq_open() — phải mở với O_WRONLY hoặc O_RDWR.
msg_ptr
Con trỏ đến buffer chứa message. Message Queue gửi raw bytes — có thể là struct, string, bất kỳ gì. Kernel copy toàn bộ msg_len bytes vào queue.
msg_len
Kích thước message, tính bằng bytes. Phải ≤ mq_msgsize của queue (đặt lúc tạo). Gửi message lớn hơn → lỗi EMSGSIZE.
msg_prio
Priority của message. Phạm vi: 0 đến 31 trên Linux (MQ_PRIO_MAX-1). Số cao hơn = ưu tiên cao hơn. Message có cùng priority được xếp theo thứ tự FIFO. Kernel tự sắp xếp — mq_receive() luôn lấy message priority cao nhất.
abs_timeout (timedsend)
Timeout tuyệt đối (CLOCK_REALTIME). Dùng clock_gettime() + cộng thêm duration. Trả về ETIMEDOUT nếu queue vẫn đầy sau timeout.
Trả về: 0 thành công, -1 lỗi. Block nếu queue đầy (blocking mode). EAGAIN nếu O_NONBLOCK và queue đầy. EMSGSIZE nếu msg_len > mq_msgsize
mq_receive() — nhận message
#include <mqueue.h> ssize_t mq_receive(mqd_t mqdes, char *msg_ptr, size_t msg_len, unsigned int *msg_prio); ssize_t mq_timedreceive(mqd_t mqdes, char *msg_ptr, size_t msg_len, unsigned int *msg_prio, const struct timespec *abs_timeout);
msg_ptr
Buffer nhận message. Kernel copy message vào đây.
msg_len
Kích thước buffer. Phải ≥ mq_msgsize của queue — không phải kích thước message thực tế. Dùng giá trị nhỏ hơn → lỗi EMSGSIZE. Pattern chuẩn: lấy mq_msgsize từ mq_getattr() rồi dùng làm buffer size.
msg_prio
Output: kernel ghi priority của message nhận được vào đây. Truyền NULL nếu không cần biết priority.
Trả về: số bytes thực sự của message (≥ 0), -1 lỗi. Block nếu queue rỗng. EAGAIN nếu O_NONBLOCK và queue rỗng
mq_close() và mq_unlink()
int mq_close(mqd_t mqdes); int mq_unlink(const char *name);
mq_close()
Đóng descriptor — giải phóng fd. Queue không bị xóa — message trong queue vẫn tồn tại. Các process khác vẫn mở được.
mq_unlink()
Xóa queue. Kernel thực sự xóa khi tất cả process đã mq_close(). Chỉ 1 process gọi unlink — thường là process tạo ra queue (server/producer).
Trả về: 0 thành công, -1 lỗi
mqd_t là file descriptor thực: Khác với SysV, POSIX mqd_t là fd hợp lệ. Có thể dùng với select(), poll(), epoll() để biết khi nào queue có message mà không cần blocking mq_receive().
Link lúc compile: Cần -lrt:
gcc sender.c -o sender -lrt
struct mq_attr và giới hạn hệ thống
Hai thông số quan trọng nhất: mq_maxmsg (bao nhiêu message) và mq_msgsize (message lớn nhất bao nhiêu bytes).
struct mq_attr — cấu trúc thuộc tính queue
struct mq_attr { long mq_flags; /* flags: 0 hoặc O_NONBLOCK */ long mq_maxmsg; /* số message tối đa trong queue */ long mq_msgsize; /* kích thước tối đa 1 message (bytes) */ long mq_curmsgs; /* số message hiện tại (read-only) */ };
mq_flags
Khi tạo queue: bị ignore (dùng oflag của mq_open). Khi đọc bằng mq_getattr(): phản ánh O_NONBLOCK. Khi set bằng mq_setattr(): thay đổi non-blocking behavior.
mq_maxmsg
Số message tối đa queue có thể chứa. Khi queue đầy, mq_send() block (hoặc EAGAIN nếu O_NONBLOCK). Trên embedded: đặt vừa đủ — mỗi slot tốn RAM = mq_msgsize bytes trong kernel.
mq_msgsize
Kích thước tối đa của 1 message (bytes). Tất cả message gửi vào queue phải ≤ giá trị này. Buffer của mq_receive() phải ≥ giá trị này. Nên đặt bằng sizeof(struct lớn nhất bạn muốn gửi).
mq_curmsgs
Chỉ có ý nghĩa khi đọc bằng mq_getattr() — cho biết số message đang chờ trong queue. Ignore khi tạo queue.
Giới hạn hệ thống — điều chỉnh cho embedded
Kernel có giới hạn mặc định cho mq_maxmsg và mq_msgsize. Nếu đặt giá trị lớn hơn, mq_open() sẽ lỗi EINVAL.
Kiểm tra và điều chỉnh limits
# Xem giới hạn hiện tại

cat /proc/sys/fs/mqueue/queues_max    # số queue tối đa (default: 256)

# Tăng giới hạn tạm thời (root):
echo 64   > /proc/sys/fs/mqueue/msg_max
echo 4096 > /proc/sys/fs/mqueue/msgsize_max

# Đặt vĩnh viễn trong /etc/sysctl.conf:

# Trên Buildroot: thêm vào /etc/sysctl.d/99-mqueue.conf

RAM kernel dùng cho queue
Kernel cấp phát trước toàn bộ RAM cho queue khi mq_open() với O_CREAT:

RAM ≈ mq_maxmsg × mq_msgsize

Ví dụ: mq_maxmsg=10, mq_msgsize=512 → ~5KB trong kernel.
Embedded với RAM ít: đặt mq_maxmsg nhỏ (4–16), mq_msgsize vừa đủ struct lớn nhất.

Tip: Dùng mq_getattr() trong code để đọc mq_msgsize thực tế rồi allocate buffer đúng kích thước — không hardcode.
Cách đặt mq_attr và đọc lại
#include <mqueue.h>

/* Tạo queue với thuộc tính tùy chỉnh */
struct mq_attr attr = {
    .mq_flags   = 0,        /* bị ignore lúc tạo */
    .mq_maxmsg  = 16,       /* tối đa 16 message đang chờ */
    .mq_msgsize = 256,      /* message lớn nhất 256 bytes */
    .mq_curmsgs = 0         /* bị ignore lúc tạo */
};

mqd_t mq = mq_open("/myqueue",
                   O_CREAT | O_RDWR,
                   0660,
                   &attr);

/* Đọc lại thuộc tính thực tế — kernel có thể điều chỉnh */
struct mq_attr actual;
mq_getattr(mq, &actual);
printf("maxmsg=%ld, msgsize=%ld, curmsgs=%ld\n",
       actual.mq_maxmsg,
       actual.mq_msgsize,
       actual.mq_curmsgs);

/* Allocate buffer đúng kích thước để nhận */
char *buf = malloc(actual.mq_msgsize);
/* ... dùng buf cho mq_receive() ... */
free(buf);

mq_notify() — async notification
Thay vì block chờ message, đăng ký để kernel tự thông báo khi queue có message mới. Không tốn CPU.
Tại sao cần mq_notify?
Blocking mq_receive(): Process ngủ chờ. Không làm gì khác được. Phù hợp cho consumer thread chuyên dụng.

Non-blocking + polling: Gọi mq_receive() liên tục trong vòng lặp. Tốn CPU, không tốt.

mq_notify() + epoll/signal: Đăng ký 1 lần. Kernel tự đánh thức khi queue có message. Process làm việc khác trong lúc chờ. Lý tưởng cho event-driven daemon.
mq_notify() — API
#include <mqueue.h> int mq_notify(mqd_t mqdes, const struct sigevent *sevp);
mqdes
Queue đã mở. Một process chỉ được đăng ký 1 notification tại một thời điểm. Process mới đăng ký sẽ thay thế process cũ.
sevp
Con trỏ đến struct sigevent — mô tả cách thông báo.

SIGEV_SIGNALGửi signal (ví dụ SIGUSR1) đến process khi có message. Đơn giản nhất nhưng hạn chế của signal handler.
SIGEV_THREADTạo thread mới và gọi callback function. Callback chạy trong thread riêng — an toàn hơn signal handler.
SIGEV_NONEKhông thông báo — chỉ "claim" queue để process khác không đăng ký được.
Truyền NULL: hủy notification đã đăng ký.
Trả về: 0 thành công, -1 lỗi. EBUSY: process khác đã đăng ký notification
Notification chỉ xảy ra 1 lần: Sau khi kernel gửi notification, registration bị xóa. Phải gọi mq_notify() lại sau mỗi lần nhận notification nếu muốn tiếp tục nhận — thường làm ngay trong callback trước khi đọc message.
Hai pattern dùng mq_notify
Pattern 1: SIGEV_SIGNAL — đơn giản, dùng với signalfd + epoll
#include <mqueue.h>
#include <signal.h>
#include <sys/signalfd.h>
#include <sys/epoll.h>

/* Block SIGUSR1 khỏi delivery bình thường (dùng signalfd thay) */
sigset_t mask;
sigemptyset(&mask);
sigaddset(&mask, SIGUSR1);
sigprocmask(SIG_BLOCK, &mask, NULL);

/* Tạo signalfd — đọc signal như file, dùng được với epoll */
int sfd = signalfd(-1, &mask, SFD_NONBLOCK | SFD_CLOEXEC);

/* Đăng ký mq_notify với SIGUSR1 */
struct sigevent sev = {
    .sigev_notify = SIGEV_SIGNAL,
    .sigev_signo  = SIGUSR1
};
mq_notify(mq, &sev);

/* Thêm signalfd vào epoll */
struct epoll_event ev = { .events = EPOLLIN, .data.fd = sfd };
epoll_ctl(epfd, EPOLL_CTL_ADD, sfd, &ev);

/* Trong epoll event loop khi sfd sẵn sàng: */
struct signalfd_siginfo sinfo;
read(sfd, &sinfo, sizeof(sinfo));  /* đọc signal info */

/* QUAN TRỌNG: đăng ký lại trước khi đọc message */
mq_notify(mq, &sev);

/* Đọc tất cả message có trong queue */
char buf[MSG_SIZE];
unsigned prio;
while (mq_receive(mq, buf, MSG_SIZE, &prio) != -1) {
    process_message(buf, prio);
}
/* mq_receive trả về -1 với EAGAIN khi queue rỗng → thoát loop */
Pattern 2: SIGEV_THREAD — callback trong thread mới, dễ dùng hơn
#include <mqueue.h>
#include <signal.h>
#include <pthread.h>

static mqd_t g_mq;

/* Callback — chạy trong thread mới do kernel tạo */
void mq_callback(union sigval sv) {
    (void)sv;
    /* QUAN TRỌNG: đăng ký lại ngay trước khi đọc */
    struct sigevent sev = {
        .sigev_notify          = SIGEV_THREAD,
        .sigev_notify_function = mq_callback
    };
    mq_notify(g_mq, &sev);

    /* Đọc tất cả message */
    char buf[512];
    unsigned prio;
    ssize_t n;
    while ((n = mq_receive(g_mq, buf, sizeof(buf), &prio)) > 0) {
        printf("Callback thread: %zd bytes, prio=%u\n", n, prio);
    }
}

int main(void) {
    g_mq = mq_open("/myqueue", O_RDONLY | O_NONBLOCK, 0, NULL);

    struct sigevent sev = {
        .sigev_notify          = SIGEV_THREAD,
        .sigev_notify_function = mq_callback,
        .sigev_value.sival_ptr = NULL
    };
    mq_notify(g_mq, &sev);

    /* Main thread tiếp tục làm việc khác — không block */
    while (1) {
        do_other_work();
        sleep(1);
    }
}
Code đầy đủ, chạy được
Hệ thống sensor với 3 loại message, priority ordering, và consumer xử lý theo thứ tự quan trọng.
msg_types.h — định nghĩa message chung
#pragma once
#include <stdint.h>
#include <time.h>

#define MQ_NAME     "/sensor_events"
#define MQ_MAXMSG   16
#define MQ_MSGSIZE  sizeof(SensorMsg)  /* buffer size cho mq_receive */

/* Priority levels — số cao = quan trọng hơn */
#define PRIO_CRITICAL  31   /* alarm, emergency */
#define PRIO_HIGH      20   /* control commands */
#define PRIO_MEDIUM    10   /* sensor data */
#define PRIO_LOW        1   /* log, heartbeat  */

/* Message type identifier */
typedef enum {
    MSG_ALARM    = 1,
    MSG_SENSOR   = 2,
    MSG_COMMAND  = 3,
    MSG_LOG      = 4
} MsgType;

/* Struct chung cho tất cả message — dùng union cho payload */
typedef struct {
    MsgType  type;
    uint32_t seq;
    struct timespec ts;

    union {
        struct { int   code;  char desc[64]; } alarm;
        struct { float temp;  float hum;      } sensor;
        struct { char  cmd[32];               } command;
        struct { char  text[80];              } log;
    } payload;
} SensorMsg;
producer.c — gửi nhiều loại message với priority khác nhau
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <mqueue.h>
#include <time.h>
#include "msg_types.h"

/* Helper: gửi message, log lỗi */
static int send_msg(mqd_t mq, const SensorMsg *m, unsigned prio) {
    int r = mq_send(mq, (const char *)m, sizeof(*m), prio);
    if (r == -1) perror("mq_send");
    return r;
}

int main(void) {
    /* Tạo queue với attr tùy chỉnh */
    struct mq_attr attr = {
        .mq_maxmsg  = MQ_MAXMSG,
        .mq_msgsize = MQ_MSGSIZE
    };

    /* Xóa queue cũ nếu còn */
    mq_unlink(MQ_NAME);

    mqd_t mq = mq_open(MQ_NAME,
                       O_CREAT | O_WRONLY,
                       0660, &attr);
    if (mq == (mqd_t)-1) { perror("mq_open"); exit(1); }

    printf("[Producer] Queue tạo: %s\n", MQ_NAME);
    SensorMsg m = {0};
    uint32_t seq = 0;

    /* Gửi 1: LOG (prio thấp) — gửi đầu tiên nhưng nhận sau cùng */
    m.type   = MSG_LOG;  m.seq = ++seq;
    clock_gettime(CLOCK_MONOTONIC, &m.ts);
    snprintf(m.payload.log.text, sizeof(m.payload.log.text),
             "System startup");
    send_msg(mq, &m, PRIO_LOW);
    printf("[Producer] Gui LOG (prio=%d)\n", PRIO_LOW);

    /* Gửi 2: SENSOR (prio trung bình) */
    m.type = MSG_SENSOR; m.seq = ++seq;
    m.payload.sensor.temp = 72.5f;
    m.payload.sensor.hum  = 58.0f;
    send_msg(mq, &m, PRIO_MEDIUM);
    printf("[Producer] Gui SENSOR (prio=%d)\n", PRIO_MEDIUM);

    /* Gửi 3: ALARM (prio cao nhất) — gửi sau nhưng nhận trước! */
    m.type = MSG_ALARM; m.seq = ++seq;
    m.payload.alarm.code = 42;
    snprintf(m.payload.alarm.desc, sizeof(m.payload.alarm.desc),
             "Temperature critical: 92C");
    send_msg(mq, &m, PRIO_CRITICAL);
    printf("[Producer] Gui ALARM (prio=%d)\n", PRIO_CRITICAL);

    /* Gửi 4: COMMAND (prio cao) */
    m.type = MSG_COMMAND; m.seq = ++seq;
    snprintf(m.payload.command.cmd, sizeof(m.payload.command.cmd),
             "EMERGENCY_SHUTDOWN");
    send_msg(mq, &m, PRIO_HIGH);
    printf("[Producer] Gui COMMAND (prio=%d)\n", PRIO_HIGH);

    mq_close(mq);
    printf("[Producer] Done — 4 messages gui vao queue.\n");
    printf("  Thu tu gui: LOG, SENSOR, ALARM, COMMAND\n");
    printf("  Thu tu nhan se la: ALARM(31), COMMAND(20), SENSOR(10), LOG(1)\n");
    return 0;
}
consumer.c — nhận theo priority, xử lý từng loại
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <errno.h>
#include <mqueue.h>
#include "msg_types.h"

static void handle_message(const SensorMsg *m, unsigned prio) {
    switch (m->type) {
    case MSG_ALARM:
        printf("  [ALARM  prio=%2u] seq=%u code=%d: %s\n",
               prio, m->seq,
               m->payload.alarm.code,
               m->payload.alarm.desc);
        break;
    case MSG_SENSOR:
        printf("  [SENSOR prio=%2u] seq=%u temp=%.1f°C hum=%.1f%%\n",
               prio, m->seq,
               m->payload.sensor.temp,
               m->payload.sensor.hum);
        break;
    case MSG_COMMAND:
        printf("  [CMD    prio=%2u] seq=%u cmd=%s\n",
               prio, m->seq,
               m->payload.command.cmd);
        break;
    case MSG_LOG:
        printf("  [LOG    prio=%2u] seq=%u %s\n",
               prio, m->seq,
               m->payload.log.text);
        break;
    }
}

int main(void) {
    /* Consumer chỉ mở — không tạo (không O_CREAT) */
    mqd_t mq = mq_open(MQ_NAME, O_RDONLY);
    if (mq == (mqd_t)-1) {
        perror("mq_open — producer chua chay?");
        exit(1);
    }

    /* Đọc mq_msgsize thực tế để allocate buffer đúng */
    struct mq_attr attr;
    mq_getattr(mq, &attr);
    printf("[Consumer] Queue: maxmsg=%ld, msgsize=%ld, curmsgs=%ld\n",
           attr.mq_maxmsg, attr.mq_msgsize, attr.mq_curmsgs);
    printf("[Consumer] Thu tu nhan (theo priority cao -> thap):\n");

    SensorMsg buf;
    unsigned prio;

    /* Đọc tuần tự — kernel trả về theo priority từ cao xuống thấp */
    while (1) {
        ssize_t n = mq_receive(mq,
                               (char *)&buf,
                               attr.mq_msgsize, /* phải >= mq_msgsize */
                               &prio);
        if (n == -1) {
            if (errno == EAGAIN) break;  /* queue rỗng */
            perror("mq_receive"); break;
        }
        handle_message(&buf, prio);
    }

    mq_close(mq);
    mq_unlink(MQ_NAME);  /* consumer xóa queue sau khi done */
    return 0;
}

Message Queue trong Embedded Linux
Pattern thực tế, ràng buộc, và so sánh với các IPC đã học để biết khi nào dùng cái nào.
Khi nào dùng Message Queue trên embedded
Phù hợp
— Event-driven system: sensor → event processor
— Cần priority: ALARM trước LOG
— Producer và consumer chạy lệch nhau về thời gian
— Message nhỏ và rõ ràng (struct ≤ vài KB)
— Cần timeout: mq_timedreceive() cho watchdog
Không phù hợp
— Dữ liệu lớn (frame ảnh, audio) → dùng Shared Memory
— Cần bidirectional theo từng request → dùng Unix Socket
— Cần nhiều consumer nhận cùng message (broadcast) → dùng Unix DGRAM socket
— Throughput cực cao → dùng Shared Memory
Tóm tắt toàn bộ IPC — bảng quyết định cuối

Pattern: event-driven daemon với mq_notify + epoll
event_daemon.c — main loop không block, xử lý nhiều fd
#include <mqueue.h>
#include <sys/epoll.h>
#include <sys/signalfd.h>
#include <signal.h>
#include <unistd.h>
#include <errno.h>
#include <stdio.h>

#define MQ_NAME  "/events"

int main(void) {
    /* ── Setup signal fd cho SIGUSR1 và SIGTERM ────────── */
    sigset_t mask;
    sigemptyset(&mask);
    sigaddset(&mask, SIGUSR1);   /* từ mq_notify */
    sigaddset(&mask, SIGTERM);   /* graceful shutdown */
    sigprocmask(SIG_BLOCK, &mask, NULL);
    int sfd = signalfd(-1, &mask, SFD_NONBLOCK | SFD_CLOEXEC);

    /* ── Mở message queue ──────────────────────────────── */
    struct mq_attr attr = { .mq_maxmsg = 16, .mq_msgsize = 256 };
    mq_unlink(MQ_NAME);
    mqd_t mq = mq_open(MQ_NAME,
                       O_CREAT | O_RDONLY | O_NONBLOCK,
                       0660, &attr);

    /* ── Đăng ký mq_notify ─────────────────────────────── */
    struct sigevent sev = {
        .sigev_notify = SIGEV_SIGNAL,
        .sigev_signo  = SIGUSR1
    };
    mq_notify(mq, &sev);

    /* ── Epoll setup ───────────────────────────────────── */
    int epfd = epoll_create1(EPOLL_CLOEXEC);

    /* Theo dõi signalfd */
    struct epoll_event ev = { .events = EPOLLIN, .data.fd = sfd };
    epoll_ctl(epfd, EPOLL_CTL_ADD, sfd, &ev);

    /* Epoll cũng theo dõi mqd_t trực tiếp (mqd_t là fd!) */
    ev.data.fd = (int)mq;
    /* Dùng EPOLLET (edge-triggered) để nhận event 1 lần */
    ev.events = EPOLLIN | EPOLLET;
    epoll_ctl(epfd, EPOLL_CTL_ADD, (int)mq, &ev);

    printf("Daemon ready, MQ: %s\n", MQ_NAME);

    struct epoll_event events[8];
    int running = 1;

    while (running) {
        int n = epoll_wait(epfd, events, 8, 5000); /* 5s timeout */
        if (n == -1) {
            if (errno == EINTR) continue;
            break;
        }
        if (n == 0) {
            printf("Heartbeat — queue idle\n"); continue;
        }

        for (int i = 0; i < n; i++) {
            if (events[i].data.fd == sfd) {
                struct signalfd_siginfo si;
                read(sfd, &si, sizeof(si));
                if (si.ssi_signo == SIGTERM) { running = 0; break; }
                if (si.ssi_signo == SIGUSR1) {
                    /* Đăng ký lại TRƯỚC khi đọc */
                    mq_notify(mq, &sev);
                }
            }
            /* Đọc tất cả message trong queue */
            if (events[i].data.fd == (int)mq ||
                events[i].data.fd == sfd) {
                char buf[256]; unsigned prio;
                while (mq_receive(mq, buf, sizeof(buf), &prio) > 0)
                    printf("  msg prio=%u: %s\n", prio, buf);
            }
        }
    }

    mq_close(mq); mq_unlink(MQ_NAME);
    close(sfd);   close(epfd);
    return 0;
}
Ràng buộc trên embedded
msg_max thường nhỏ: Kernel mặc định msg_max=10. Nếu code cần mq_maxmsg=16, phải tăng trước khi chạy. Thêm vào startup script: echo 64 > /proc/sys/fs/mqueue/msg_max. Không làm được → mq_open() lỗi EINVAL.
mqueue filesystem cho debug: Trên Buildroot thêm vào /etc/fstab: mqueue /dev/mqueue mqueue defaults 0 0. Sau đó ls /dev/mqueue/ thấy tên queue, cat /dev/mqueue/events thấy thông số. Rất hữu ích khi debug queue không được xóa sau crash.
Queue orphan sau crash: Nếu producer crash mà không mq_unlink(), queue còn tồn tại. Lần sau mq_open() với O_CREAT | O_EXCL sẽ lỗi EEXIST. Giải pháp: bỏ O_EXCL và gọi mq_unlink() rồi tạo lại, hoặc trong startup script: rm -f /dev/mqueue/myqueue.
Tài liệu này kết thúc series IPC. Bạn đã học đủ 4 loại IPC cốt lõi:
1. Pipe & FIFO — nền tảng, đơn giản nhất
2. Unix Domain Socket — bidirectional, nhiều client, fd passing
3. Shared Memory — zero-copy, throughput cao nhất, cần semaphore
4. Message Queue — priority ordering, decoupled timing, async notify

Bước tiếp theo thực tế: chọn 1 trong 4 loại và viết một ứng dụng nhỏ hoàn chỉnh trên STM32 Linux hoặc Buildroot để trải nghiệm thực tế.
